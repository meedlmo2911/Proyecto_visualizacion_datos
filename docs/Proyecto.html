Se convirtió el dataframe de pandas a un geodataframe mediante geopandas y se ponderaron los datos por energía (transformando la magnitud de cada terrremoto a energía). Se asignó cada terremoto a su país correspondiente dependiendo de sus coordenadas gracias a geopandas. Primero, los terremotos que deberían entrar en las fronteras de un país pero que se encontraban en el océano, no contaron en la suma final, esto debido a que el mapa de resolución media no toma en cuenta los límites marítimos. Adicionalmente, al ser de resolución media países muy pequeños también tienen ciertos errores. Finalmente si se toman en cuenta los países con “cero” terremotos dentro de la escala, esta escala cambia y se vuelve más difícil clasificar los países con más terremotos.
    • Mapa de calor de terremotos en Sudamérica

Desde el año 1900 se han registrado 3 terremotos con una magnitud mayor a 5, mientras que terremotos de magnitud menor a 5 son más comunes. Pero la mayoría de estos se registra muy cerca de las fronteras con Brasil, Argentina o en el estuario del río de la plata.

Hay 593 valores nulos de profundidad, estos valores se mantendrán debido a que la evolución temporal de los terremotos se realizará con las columnas de fecha, magnitud y coordenadas y es conveniente para el análisis conservar los registros más antiguos aunque no se tenga el dato de profundidad.

Mapa global de epicentros

El objetivo de esta visualización es poder identificar de manera fácil los países que tienen mayor densidad de terremotos. Para esto se ha propuesto una visualización usando geopandas haciendo uso de las coordenadas para posicionar los diferentes terremotos.

Para realizar la visualización se utilizó el registro de geopandas de resulución media de 1:50m. Se convirtió el dataframe de pandas a un geodataframe mediante geopandas y se ponderaron los datos por energía (transformando la magnitud de cada terrremoto a energía). Se asignó cada terremoto a su país correspondiente dependiendo de sus coordenadas gracias a geopandas. Finalmente se agruparon y se sumaron las energías de los terremotos en cada país y se graficaron los resultados en un mapa de colores. Realizando esto se pueden identificar varias fuentes de errores. Primero, los terremotos que deberían entrar en las fronteras de un país pero que se encontraban en el océano, no contaron en la suma final, esto debido a que el mapa de resolución media no toma en cuenta los límites marítimos. Adicionalmente, al ser de resolución media países muy pequeños también tienen ciertos errores. Finalmente si se toman en cuenta los países con “cero” terremotos dentro de la escala, esta escala cambia y se vuelve más difícil clasificar los países con más terremotos.

Mapa de calor de terremotos en Sudamérica

Particularmente para Sudamérica se realizó el mismo tratamiento con geopandas del dataframe. Para Uruguay, se tuvieron muy pocos registros, esto se debe a lo siguiente. Desde el año 1900 se han registrado 3 terremotos con una magnitud mayor a 5, mientras que terremotos de magnitud menor a 5 son más comunes. Pero la mayoría de estos se registra muy cerca de las fronteras con Brasil, Argentina o en el estuario del río de la plata.

import geopandas as gpd
import pandas as pd
import matplotlib.pyplot as plt

# =========================
# 1. Cargar mapa de países
# =========================
url = "https://naturalearth.s3.amazonaws.com/50m_cultural/ne_50m_admin_0_countries.zip"
world = gpd.read_file(url)

# =========================
# 2. Limpiar datos
# =========================
data = data.dropna(subset=["latitude", "longitude", "mag"])


# =========================
# 3. Convertir a GeoDataFrame
# =========================
gdf = gpd.GeoDataFrame(
    data,
    geometry=gpd.points_from_xy(data.longitude, data.latitude),
    crs="EPSG:4326"
)

# =========================
# 4. Spatial join (asignar país)
# =========================
gdf_world = gpd.sjoin(gdf, world, how="left", predicate="intersects")

# =========================
# 5. PONDERACIÓN POR MAGNITUD
# =========================
# Convertir magnitud a energía
gdf_world["energia"] = 10 ** (1.5 * gdf_world["mag"])

# Agrupar por país
obs_por_pais = (
    gdf_world.groupby("NAME")["energia"]
    .sum()
    .reset_index(name="energia_total")
)

# Unir con mapa
world = world.merge(obs_por_pais, on="NAME", how="left")
world["energia_total"] = world["energia_total"].fillna(1)

# =========================
# 7. Graficar heatmap
# =========================
fig, ax = plt.subplots(figsize=(12,6))

world.plot(
    column="energia_total",
    cmap="berlin",
    linewidth=0.5,
    edgecolor="white",
    legend=True,
    norm=colors.LogNorm(),
    ax=ax
)

ax.set_title("Energía sísmica por país (escala logarítmica)")
ax.axis("off")

plt.show()

# 3. CONVERTIR A GEODATAFRAME
# =========================
gdf_quakes = gpd.GeoDataFrame(
    data,
    geometry=gpd.points_from_xy(data["longitude"], data["latitude"]),
    crs="EPSG:4326"
)

# =========================
# 4. CARGAR MAPA DE PAÍSES (ALTA CALIDAD)
# =========================
url = "https://naturalearth.s3.amazonaws.com/50m_cultural/ne_50m_admin_0_countries.zip"
world = gpd.read_file(url)

# normalizar nombres de columnas
world.columns = world.columns.str.lower()

# =========================
# 5. SPATIAL JOIN (asignar terremotos a países)
# =========================
gdf_join = gpd.sjoin(gdf_quakes, world, how="inner", predicate="within")

# =========================
# 6. AGRUPAR ENERGÍA POR PAÍS
# =========================
energia_por_pais = gdf_join.groupby("name")["energia_sismica"].sum().reset_index()

# =========================
# 7. UNIR CON MAPA
# =========================
world = world.merge(energia_por_pais, on="name", how="left")
world["energia_sismica"] = world["energia_sismica"].fillna(0)

# =========================
# 8. FILTRAR SUDAMÉRICA
# =========================
south_america = world[world["continent"] == "South America"]
